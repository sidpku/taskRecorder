# TaskRecorder - 时间打卡记录工具

## 项目概述

个人时间管理工具，通过"打卡"记录时间点，自动将相邻打卡配对形成时间段，帮助用户回顾每天的时间使用情况。

## 领域术语

| 术语 | 定义 |
|------|------|
| **打卡（Punch）** | 记录一个时间点的动作。相邻两个打卡自动形成一个时间段。 |
| **时间段（TimeEntry）** | 由两个相邻打卡点构成的时间区间，包含开始时间、结束时间和描述。 |
| **标签（Tag）** | 预设的快捷描述文字。点击标签等同于打卡并填入该标签文字作为描述。 |
| **描述（Description）** | 对一个时间段内所做事情的文字记录。可通过标签快捷填入，也可手动输入。 |
| **日开始打卡** | 每天第一次打卡，标记为"一天的开始"，可填描述，作为后续时间段的起点。 |

## 核心行为规则

1. **回顾式描述**：打卡时描述的是"从上次打卡到现在这段时间做了什么"。
2. **标签即快捷描述**：标签不是分类维度，而是预写好的描述文字的快捷方式。
3. **多标签拼接**：一次打卡可选多个标签，拼接在一起作为描述。
4. **第一次打卡**：标记为一天的开始，可填描述（如"到公司"），作为时间段起点。

## 功能边界

### 包含

- 打卡记录时间点
- 手动输入描述 或 点击标签快捷填入
- 多标签多选拼接
- 实时显示"上次打卡时间"和"已用时 X 分钟"
- 今日时间段列表（时间线）
- 编辑和删除已有打卡记录（修改描述、调整时间、删除）
- 标签管理（独立区域：添加、编辑、删除）
- 标签自动生成颜色，支持手动修改
- CSV 导出（每行一个时间段：开始时间 | 结束时间 | 时长 | 描述）
- CSV 导出支持日期范围选择 + 快捷选项（今天、本周）

### 不包含

- 多设备同步
- 用户认证/多用户
- 统计分析图表（MVP 阶段）

## 技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 前端框架 | Vite + React | 现代、快速、生态成熟 |
| 后端框架 | Node.js + Express | 全栈 JS，轻量 API |
| 数据存储 | JSON 文件 | 人眼可读，个人工具数据量小 |
| 数据组织 | 按天分文件（`data/2026-05-15.json`） | 天然按天隔离，文件不膨胀 |
| 标签存储 | 独立 `data/tags.json` | 标签跨天共用 |
| 页面结构 | 单页面分区 | 打卡追求快速操作，避免页面跳转 |
| 标签管理 | 弹窗 | 独立管理，不干扰打卡主流程 |
| CSV 导出 | 时间段模式 | 直接呈现最终想看的信息 |

## 数据结构示例

### 打卡记录（`data/2026-05-15.json`）

```json
[
  { "id": "uuid1", "time": "2026-05-15T09:00:00", "description": "到公司" },
  { "id": "uuid2", "time": "2026-05-15T10:30:00", "description": "开会, 需求讨论" },
  { "id": "uuid3", "time": "2026-05-15T12:00:00", "description": "写代码" }
]
```

### 标签（`data/tags.json`）

```json
[
  { "id": "uuid-a", "name": "开会", "color": "#4A90D9" },
  { "id": "uuid-b", "name": "写代码", "color": "#7ED321" },
  { "id": "uuid-c", "name": "休息", "color": "#50E3C2" }
]
```

### CSV 导出示例

```
开始时间,结束时间,时长(分钟),描述
2026-05-15 09:00,2026-05-15 10:30,90,开会, 需求讨论
2026-05-15 10:30,2026-05-15 12:00,90,写代码
```
